Настройки системы

Параметры LLM, API-ключи и системные промпты хранятся в базе данных и кэшируются в Redis. По умолчанию используется DeepSeek (совместимый с OpenAI SDK, base_url=https://api.deepseek.com).

В продакшене ключи в БД следует шифровать at rest (например, cryptography.fernet).

Ключи API

Опрос GET /updates у MAX без публичного HTTPS. В продакшене выключайте и используйте Webhook; по правилам MAX одновременно не использовать оба способа.

Сколько последних реплик (user/assistant) подмешивать в запрос к модели; действует для MAX, веб-чата и голоса.

Подстрока в тексте (например @id…_bot). В групповых чатах бот обрабатывает сообщение только при наличии этого упоминания; в личных диалогах правило не применяется.

Числовой chat_id группы MAX (как в панели «Боты»). Если session_id совпадает с этим значением, к базовому промпту консультанта добавляется текст ниже. Оставьте пустым, если не нужно.

Вставляется в системное сообщение после основного промпта консультанта и инструкций по CRM, до дополнения для текстовых ботов.

Голос: SaluteSpeech (Сбер)

При VOICE_STT_PROVIDER=salutespeech и/или VOICE_TTS_PROVIDER=salutespeech в .env ключ можно задать здесь (или переменная SALUTESPEECH_AUTH_KEY в окружении контейнера).

Промпты

Добавляется к системному промпту после основного промпта консультанта и инструкций по CRM. Используйте для правил формата ответа в мессенджере (длина, эмодзи, обращение на «вы», без Markdown и т.д.). Не применяется к голосовому каналу и к полю выше по отдельности — только там, где сценарий вызывается как бот MAX/Telegram.
